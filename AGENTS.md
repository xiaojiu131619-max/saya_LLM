# Agent_LLM 项目规则

## 语言与文案

- 本项目以中文为第一语言。
- 除必要技术词汇外，所有菜单、按钮、说明、提示、错误信息、设置项、UI 文案和文档新增内容都必须优先使用中文。
- 可保留英文的必要词汇包括：命令名、参数名、API 名、协议名、库/框架/产品名、模型名、文件扩展名、日志原文，以及行业通用缩写，例如 `GGUF`、`llama.cpp`、`ComfyUI`、`API key`、`ctx`、`ngl`、`KV cache`、`Flash Attention`、`RoPE`。
- 当英文技术词汇出现在 UI 中时，优先配中文解释或中文主标签，例如“上下文长度（ctx）”“GPU 卸载（ngl）”。
- 不要新增大段英文说明文案；若引用外部错误或日志，可保留原文，但面向用户的解释必须用中文。

## 参数与排障

- 不要静默修改默认加载或推理参数。用户的默认值是排障基线，`ctx`、`ngl`、KV、batch、parallel、max token、reasoning 等行为必须保持显式、可复现，除非用户明确要求更改默认值。
